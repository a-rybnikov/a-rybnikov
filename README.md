**Независимый исследователь · безопасность агентных и высоконагруженных систем · непрерывность ИИ**

Я строю и проверяю на прочность системы, где автономные ИИ-агенты и сервисы получают
доступ к **криптокошелькам (приватным ключам)**, **закрытым секретным данным** и работают
**под высокой нагрузкой**. Чтобы такой системе можно было доверять, нужно знать, как она
ломается — и уметь это измерять.

### Конвейер автоматизированного тестирования безопасности

Шесть небольших инструментов с тестовым покрытием, проходящих **всю поверхность**
автономной системы — от разведки до непрерывного мониторинга в проде. Пять прощупывают
систему, шестой её охраняет.

1. **[mcpx](https://github.com/a-rybnikov/mcpx)** — *разведка.* Чёрный-ящик-инвентаризация
   поверхности сервиса: транспорт, рукопожатие, перечень доступных инструментов, ресурсов и
   полномочий, отпечаток, оценка защищённости.
2. **[needler](https://github.com/a-rybnikov/needler)** — *фаззинг.* Атака на входные данные
   найденных инструментов по их схемам — мутации аргументов против каждого `inputSchema`.
3. **[overreach](https://github.com/a-rybnikov/overreach)** — *избыточные полномочия.* Оценка,
   не даёт ли система больше прав, чем необходимо (OWASP LLM-08): функциональность /
   разрешения / автономность. Доступ агента к кошельку — канонический случай.
4. **[ghostwrite](https://github.com/a-rybnikov/ghostwrite)** — *устойчивость памяти.*
   Подсаживает «отравленную» запись и проверяет, переживает ли она границу сессии и меняет
   ли поведение системы — атака на состояние.
5. **[snare](https://github.com/a-rybnikov/snare)** — *непрямые инъекции.* Ловушки
   (indirect prompt-injection) для агентов, работающих с браузером и внешними данными, с
   canary-подтверждением фактического срабатывания.
6. **[shadow](https://github.com/a-rybnikov/shadow)** — *мониторинг дрейфа (blue team).*
   Поведенческий baseline, который ловит компрометацию в проде: подмену модели, новый
   незаявленный инструмент, тихий обход правил, переход «отказал → выполнил».

Основано на оплачиваемой работе — **аудит web3 / on-chain** (агент, держащий ключи, — это
эталонная находка по избыточным полномочиям) и **HackerOne**; сейчас углубляюсь в Solidity.
Родословная инструментов: [garak](https://github.com/NVIDIA/garak) /
[PyRIT](https://github.com/Azure/PyRIT).
